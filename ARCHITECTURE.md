> 이 파일을 Claude에게 전달하면 프로젝트의 전체 맥락을 복원할 수 있습니다.

## 1. 프로젝트 개요

- **목적**: 사용자+LLM 하이브리드 투자 비서 (네비게이터)
- **투자 철학**: 터틀 트레이딩 주축, Minervini/O'Neil 보조, "예측하지 말고 대응"
- **핵심 원칙**:
  - 절대적 기준만 추천 (차선책 금지, 없으면 "없음")
  - 1% 리스크 룰 (단일 거래 최대 손실 = 총자산의 1%)
  - 차트 색상은 중립 톤만 (빨강/초록/파랑 금지)
  - 거래하지 않는 것도 포지션

## 3. 시스템 아키텍처 — 4층 스캔 + 6대 모듈

### 3-1. 4층 스캔 구조

```
L0  글로벌 자산 RS 비교  (월 1회)
    ├ 한국주식, 미국주식, 금, 원유, 구리, 채권, BTC, 환율
    ├ 6개월 가중 RS (최근 3개월 2배 + 3~6개월 1배)
    └ 상위 2~3개 자산군만 L1 진입
        ↓
L1  자산군 내 섹터 RS  (2주 1회)
    ├ 한국: 테마 ETF 20~30개 RS 랭킹
    ├ 미국: 섹터 ETF (XLK, XLE, SMH 등)
    └ 상위 3~5개 섹터만 L2 진입
        ↓
L2  섹터 내 종목 선정  (신호 발생 시)
    ├ 20일/55일 신고가 돌파 (터틀 진입)
    ├ VCP 수축 확인 (질 필터, 좁은 손절폭)
    ├ Stage 2 정배열 (Trend Template)
    ├ 거래량 동반 확인
    └ 모든 조건 충족 시에만 매수 신호
        ↓
L3  실행 (원화·국내 ETF 우선)
    ├ ATR 기반 1% 리스크 사이징
    ├ Trailing Stop + Time Stop (2주 본전이면 정리)
    └ 결과 → Downloads 폴더 저장
```

### 3-2. 6대 모듈

| 모듈 | 역할 | 데이터 소스 |
|------|------|-------------|
| **M1 심리** | 뉴스 감성 + 비경제 커뮤니티 언급 빈도 + 뉴스 vs 시장반응 괴리 | 크롤러 (추후) |
| **M2 매크로** | 미/한 통화량·적자·인플레·정책 요약 | FRED, 한은 API (추후) |
| **M3 RS** | 글로벌 자산→섹터 RS 랭킹 | yfinance, pykrx |
| **M4 분석** | 터틀 돌파 + VCP + 한국 백데이터 검증 | pykrx, yfinance |
| **M5 포트** | 보유자산 기반 매일 판단, 저널 | portfolio.json |
| **M6 차트** | 중립 색상 차트 (흑백/그레이만) | matplotlib/plotly |

## 6. 매매 규칙 요약

### 진입 조건 (모두 충족 시에만)
- [ ] L0: 해당 자산군 RS 상위 3
- [ ] L1: 해당 섹터 RS 상위 5
- [ ] 시장 체제: 가격 > MA200 AND MA50 > MA200 (정배열)
- [ ] 20일 또는 55일 신고가 돌파
- [ ] VCP 수축 확인 (좁은 손절폭 가능)
- [ ] 거래량 동반

### 포지션 사이징
```
1거래 최대 리스크 = 총자산 × 1%
주당 리스크 = 2 × ATR(20)
매수 수량 = (총자산 × 1%) ÷ (2 × ATR)
```

### 청산 조건 (하나라도 해당 시)
- Trailing Stop 이탈 (최고가 - 2×ATR)
- N일 저점 이탈 (System1: 10일, System2: 20일)
- Time Stop: 2주 후 ±2% 미만이면 정리

### 절대 금지
- 조건 미충족 시 "차선책" 매수
- 손절가 없는 진입
- 물타기 (평균단가 낮추기 위한 추가매수)
- 뉴스/감정 기반 뇌동매매
