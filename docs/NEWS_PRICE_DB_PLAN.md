# 30년+ 뉴스-가격 반응 데이터베이스 — 프로젝트 계획서

> 별도 프로젝트로 완성 후, 추세추종 블룸버그 터미널에서 참조.
> "끔찍한 뉴스에 시장이 상승하면 그것은 중요한 것을 암시한다."

---

## 1. 목적

**역사적으로 중대 뉴스가 나왔을 때 시장이 어떻게 반응했는지** 데이터베이스를 구축.
→ 현재 발생하는 뉴스와 유사한 과거 사례를 즉시 검색하여 참고.

### 활용 시나리오
```
오늘 뉴스: "연준 0.5% 긴급 인하"
    ↓
DB 검색: 과거 긴급 인하 사례 (2001, 2007, 2008, 2020)
    ↓
결과: 긴급 인하 후 30일 평균 수익률: -8.3%
      단, 인하 후 VIX < 30이면: +4.2%
    ↓
→ "긴급 인하 자체는 하락 신호지만, 공포가 이미 해소된 상태면 반등"
```

## 3. 데이터베이스 스키마

### events 테이블
```
id          INTEGER PRIMARY KEY
date        DATE            -- 이벤트 발생일
category    TEXT            -- "fed", "inflation", "geopolitical", "trade", ...
subcategory TEXT            -- "rate_cut", "emergency_cut", "qe_start", ...
title       TEXT            -- "연준 0.5% 긴급 인하"
title_en    TEXT            -- "Fed Emergency 50bp Rate Cut"
description TEXT            -- 상세 설명
severity    INTEGER (1~5)   -- 시장 충격 수준 (5=최대)
surprise    INTEGER (-2~+2) -- 예상 대비 서프라이즈 (-2=매우 비둘기, +2=매우 매파)
source      TEXT            -- 출처
```

### market_reactions 테이블
```
event_id    INTEGER REFERENCES events(id)
asset       TEXT            -- "SPX", "KOSPI", "Gold", "WTI", "VIX", ...
price_before REAL           -- 이벤트 전일 종가
price_day0  REAL            -- 당일 종가
price_day1  REAL            -- 1일 후
price_day5  REAL            -- 5일 후
price_day20 REAL            -- 20일 후
price_day60 REAL            -- 60일 후
change_day0 REAL            -- 당일 변화율 (%)
change_day5 REAL            -- 5일 변화율 (%)
change_day20 REAL           -- 20일 변화율 (%)
change_day60 REAL           -- 60일 변화율 (%)
vix_at_event REAL           -- 이벤트 시점 VIX
```

### context 테이블 (이벤트 당시 시장 상태)
```
event_id    INTEGER REFERENCES events(id)
spx_ma200   TEXT            -- "above" / "below" (정배열/역배열)
vix_level   TEXT            -- "low(<15)", "normal(15-25)", "high(25-35)", "extreme(>35)"
rate_trend  TEXT            -- "hiking", "holding", "cutting"
yield_curve TEXT            -- "normal", "flat", "inverted"
```

## 5. 구현 단계

### Step 1: 스키마 + 가격 데이터 자동 수집 (1주)
- SQLite DB 생성
- yfinance로 30년 가격 데이터 일괄 다운로드
- 자동화: 이벤트 날짜 입력 → 전후 가격 자동 계산

### Step 2: 핵심 이벤트 수동 입력 (2~3주)
- 상위 100건 먼저 (가장 시장 충격 큰 이벤트)
- 카테고리·서프라이즈·심각도 태깅
- context 테이블 (당시 시장 상태) 채움

### Step 3: 분석 엔진 (1~2주)
- 쿼리: "과거 긴급 인하 후 30일 S&P 평균 수익률"
- 쿼리: "VIX > 30 상태에서 전쟁 발발 후 금 가격 변화"
- 조건부 통계: 이벤트 유형 × 당시 시장 상태 → 후속 수익률

### Step 4: 터미널 연동 (1주)
- 대시보드에 "유사 과거 사례" 패널 추가
- 오늘 뉴스와 가장 유사한 과거 이벤트 자동 매칭
- 후속 수익률 통계 표시

## 6. 터미널에서 표시 방식

```
┌──────────────────────────────────────────────┐
│  뉴스-가격 과거 사례 (News-Price DB)          │
├──────────────────────────────────────────────┤
│  현재 상황: 이란전쟁 + 연준 동결 + VIX 22     │
│                                               │
│  유사 사례:                                    │
│  1. 2022 러-우 전쟁 (VIX 30)                  │
│     S&P: -8% (20일) → +12% (60일)            │
│     금: +6% (20일)                             │
│     유가: +30% (20일) → +5% (60일)            │
│                                               │
│  2. 2003 이라크전 (VIX 28)                    │
│     S&P: -3% (20일) → +20% (60일)            │
│     금: +2% → -5%                             │
│                                               │
│  통계 (전쟁+VIX 20~30 조건):                   │
│     S&P 60일 후 평균: +8.5% (4건 중 3건 상승)  │
│     금 60일 후 평균: +3.2%                     │
└──────────────────────────────────────────────┘
```
